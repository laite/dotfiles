Main Structure

All pages in koirakamut.com follow the same basic structure, where the sidebar is on the right, and between header and footer there is main section, which can then be divided into multiple main section blocks (Figure1).

If user makes the window horizontally smaller, the sideBar jumps over the main section, thus resulting in a slim, yet functional layout. (Figure2)

Javascript usage

Javascript is only used in two ways on Koirakamut:

  1. Tell the user that search/login is not possible (through simple alert()- popup)
  2. To toggle visibility of specific items either immediately or smoothly
I have intentionally kept the page light and resisted the urge to put all sorts of whistles and sirens around.

Usability Analysis

Overall design

Koirakamut.com is designed to have a simple, clean layout which stays static and consistent from page to page.

On top there's a main banner/logo which always takes user back to index.html. IMHO logo should always take user to main page, see bad example from Tampereen joukkoliikenne

On index page there's an login row below banner, but other than that pages' main content starts immediately. This allows user to start finding relevant information right away. At first I thought there could have been some javascript which would rotate pretty pictures but soon found out that pretty pictures are quite unpractical. No beating around the bush!

Navigation bar (in its 'large' form) consists of links, quicksearch and a cute puppy. Puppy is there because puppies are cute. On smaller screens puppy, being non-relevant, disappears entirely.

Main section is designed to be quite modular with 'mainSectionBlocks', so there could in theory be a whole lot of things, if necessary.

Applied Gestalt laws

  • Groupings

    Items with same agenda are nested into groups to make it easier to separate them from other items in the page. This kind of placement also embraces Gestalt design laws such as Continuance and Proximity

  • Similarity

    All pages use same basic layout described in Technical Specifications, but also smaller blocks within the layout are designed to have similarity. For instance, time/date - parts are the same color/format/size for both the dog profiles and news items.

    There are also some cases where background color is used to bring similarity to user mind: login form and search information share common style called 'informativeBackground' - this should make it clear for the user that there's something which optionally could be somehow used.

  • un-Similarity

    One thing that (aware) user probably notices right away is that the navigation bar really stands out from the rest of the page - it's quite heavily rounded corners and blueish background makes it easy for user to immediately notice that 'this is the thing I need'. Even when user is on a small screen and navigation bar changes place, it's color and shape stay the same.

Final Thoughts

Learning to create Web Pages is quite fun, although I was bit annoyed by the amount of hardcoding within the page, but that'll surely diminishes once proper methods are learned (e.g. use of external html's for some repetitive blocks, reading text items from databases). Also, CSS seemed at times a bit cumbersome (where are my variables!) but thankfully there are a ton and a half of tutorials in the internet.

Koirakamut uses javascript really sparingly, mainly because of my dislike to flashing and/or bouncing things in the internet. My initial attempt was to build a clean, sparse layout with pretty colors and I think that succeeded well enough. Technical demos can be found in the TOC.

Of course there are still all sorts of tweakings and fixings I'd like to do for the page - but then again, isn't there always... :)
