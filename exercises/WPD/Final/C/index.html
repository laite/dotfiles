Introduction

Developing web page is a tedious task. Not only should one pay attention on the contents, but also the visual design, user experience and the quality of code.

This page is a minute approach to Web Design, where I try to talk about some general principles and pointers for creating a good web site . Everything on this page is to be treated as a personal opinion, since I have no real education on the subject.

I have included a small glossary for some problematic terms or just to add some advanced discussion without disrupting the user experience in the current page. There are also some external links about the subject.

General Principles

There are some General Principles that every good web developer should take into account. Not only should web designer try to create a stunning, visually appealing page, he/she must also be able to create it so that people who have never seen the page before would immediately feel at ease with it.

Gestalt Design Laws

Gestalt design laws are a set of principles that can be used as a framework when designing user interfaces. Few of the most common are listed here.

  • Closure: the mind supplies the missing pieces in a composition
  • Continuance: the eye continues in the direction it is going
  • Similarity: what an items looks like and how that effects gestalt
  • Proximity: where items are in relationship to each other and how that effects gestalt
  • Alignment: lining up objects to organize and form groups

Laws of Klinga

Of course, Gestalt is just the beginning. Cue laws of Klinga.

  • There is nothing wrong with boring
  • Items really shouldn't change place after they are initialized
  • Content is always more important than you think

Using The Web

Web usability could be described as 'get out of my way' - people do not want to use web pages, they want to click largest, brightest button and recieve the exact information they're looking for (preferably with super cool audiovisual effect storm, but that's another story).

Some Do's and Dont's

These are some things that one should consider before/while creating wep pages.

  • Who are you designing for

    It's quite different to create web page for your local boyscout branch or eager, yet narrowminded advertisement company. Designer must know the audience and write to them.

  • Style vs. usability

    Some web pages are so cool they throw usability straight to hell. "Colored links? Bah. Let's have more cyb0rkittahs instead."

  • Users are always busy

    Walls of text have a bad reputation. For some reason users in the Internet have practically no time at all to read things. They want everything instantly or they move away. If you really want to write, make sure that your page is serene, peaceful and well structured so nothing takes away users attention. It also helps if you actually have something to say.

  • Users know better

    Whatever you do, it's probably wrong. So don't be afraid to copy draw inspiration from the most popular web sites in the world - that's how people mostly 'use their Internet' anyway and that's how they expect things to work.

Advanced Tips

NOTE: These tips are taken from Research-Based Web Design & Usability Guidelines and seem a bit outdated.

The Homepage

The homepage can invite or turn away; usually one can instantly see whether he/she likes or hates the service he/she has stumbled upon. First impression is crucial.

Generally The Homepage should be source for latest news (shortly) and a portal to all the important services on the site. Large promotion images / offers are usually allowed and tolerated.

Page Layout

It's probably a good idea to put important things first on page, or at least include page-specific Table of Contents if there's a lot of text or other items. One should also pay attention to the length of lines and paragraphs.

Main focus should always be on readability and overall clarity of layout; do not try to have too much things on the same page. Be consistent.

Despite Research-based Web Design... I would say that there is usually nothing wrong with having much white space on layout - as long as the main content fulfills Gestalt idea of proximity. Obviously this depends a whole lot on the overall layout.

Navigation

Be very cautious with opening new browser windows - it's usually really annoying and unnecessary. If possible, user should always be able to use browser's 'back' button to return from last link (many javascripts break this)

Use anchor links on long web pages. It's usually a good idea to put high level links on topmost or at left side of screen and lower level under/beside them. Page should also always give user simple feedbak about where he/she is currently.

One shouldn't create any "navigation-only" type pages, there are a lot of better ways to implement this kind of navigation (unless there's a lot of different links, see e.g. debian.org sitemap).

Site maps as such are (IMO) a bit old-fashioned, and it would be better to categorize site so that they wouldn't be even needed. Obviously this is not always possible. Breadcrumb Navigation is great alternative!

Headings etc.

Don't dismiss page titles, use unique and clear headings.

Categorize elements on the page and use different levels for headers accordingly.

Text Appearance

Don't go wild with the fonts. Stay with the basic sans or sans-serif. Make sure there is always enough contrast between text and background. Format similar items consistently.

Embhasizing things is a good thing, but only if it's used sparingly.

Graphics And Multimedia

Use images sparingly, consistently and without disturbing the overall experience. One should be really careful about videos/flash - animations. And never start any audio without asking user.

Actual (research etc.) data is usually nice to have as a visual element. It's usually safe to have graphical elements resemble real world, so users know immediately their purpose.

Even though bandwidths tend to be quite large these days, there's nothing wrong in providing thumbnail-sized images when it's not necessary to load whole large one.
